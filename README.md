# MeiBeautyOfficial - 保養品牌購物車電商網站

這是一個獨立開發的電商購物車專案。本專案以保養品牌為背景，採用前後端分離/半分離（Thymeleaf/RESTful API）的架構開發，深入實作了電商核心的購物流程、會員權限驗證、資料庫交易管理以及後端自動化排程。

## 🛠 開發技術與工具

### 後端核心技術
後端核心： Java 17+, Spring Boot
資料庫 ORM： Spring Data JPA / Hibernate
安全與權限： Web Session / Cookie 會員登入驗證、設計權限管控確保特定操作。
資料庫： MySQL (包含資料表建立、多條件複合搜尋)
前端互動與儲存： JavaScript, Ajax, DataTable, LocalStorage 購物車狀態暫存

### 資料庫
* **關聯式資料庫：** MySQL (包含資料表正規化設計、高效能索引查詢與條件搜尋)

### 前端互動與儲存
* **網頁渲染：** HTML5, CSS3, Thymeleaf 模板引擎
* **動態互動：** JavaScript, Ajax, DataTable
* **客戶端儲存：** LocalStorage, Cookie, Session 整合應用

## 🚀 核心功能與設計亮點

### 1. 完整電商購物與結帳流程
* **商品瀏覽與搜尋：** 支援多條件複合搜尋，並結合前端 DataTable 實現流暢的商品展示。
* **購物車與收藏機制：** 具備商品加入/刪除購物車、動態計算金額功能，並提供收藏清單（Wishlist）功能。
* **下單與交易紀錄：** 串接 MySQL 資料庫，於使用者結帳時完成訂單寫入，確保下單紀錄的資料完整性。

### 2. 進階會員系統與安全驗證
權限管控機制： 實作攔截器（Interceptor/Filter），確保使用者必須在特定登入狀態下，才能執行下單結帳等敏感操作。

### 3. 資料自動化管理與系統優化 (※ 面試核心加分項)
* **輕量化前端購物車：** 運用客戶端 LocalStorage 儲存使用者選購的商品 ID，大幅降低伺服器（Server）在初期的儲存負擔與連線壓力。
* **RESTful API 設計：** 後端交易與下單： 結帳時，前端將商品 ID 清單透過 RESTful API 傳遞至後端，由後端整合 MySQL 資料庫進行動態商品資訊補完，並安全地完成訂單寫入。

## 🗺 未來優化方向 (Roadmap)

為了進一步提升系統的擴充性與安全性，預計下一階段將進行以下架構升級：

1. **購物車架構升級（預計導入 Quartz 排程）：**
   目前購物車暫存於前端 LocalStorage。未來規劃將未結帳的購物車資料改由後端資料庫或 Redis 統一管理，以便實作「跨裝置同步購物車」功能；屆時將導入 **Quartz 框架**，設定自動化定時任務，定期清理超過 7 天未結帳的廢棄購物車資料，優化資料庫效能。

2. **驗證機制升級（預計導入 JWT）：**
   目前採用傳統的 Session 驗證。隨著未來若有前後端完全分離或微服務化的需求，預計將登入機制改為 **JWT (JSON Web Token)** 無狀態驗證，並整合第三方登入，提升系統的擴充性。
